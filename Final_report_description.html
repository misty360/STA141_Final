Overview

The goal of this final project is to develop a prediction model for the incidence and location of notable wildfires in the United States (link). You will use historical wildfire records to estimate the probability that a wildfire will occur in a given geographic area during a future month.

Data and background

You can find and download a subset of historical wildfire records derived from the Kaggle dataset “188 Million US Wildfires”. The dataset includes wildfire ignition locations, discovery dates, containment dates, and final burned area across the contiguous United States. You are encouraged, but not required, to use the full dataset from the original source.

Wildfires vary widely in size, and the vast majority of recorded fires are very small and quickly contained. Modeling all recorded fires would lead to extreme class imbalance and would obscure the dynamics of fires that are operationally and scientifically meaningful.

To address this, you may filter the raw wildfire records to retain only “notable” fires. A recommended cutoff is to retain only the wildfires with final burned area \(\geq\) 300 acres. This threshold is consistent with operational wildfire reporting standards used by U.S. fire management agencies, where large wildland fires are commonly defined as: \(\geq\) 100 acres in timber fuel models, or \(\geq\) 300 acres in grass or brush fuel models

Alternative Cutoffs. You may choose an alternative cutoff if you clearly justify your choice. Any alternative cutoff must be clearly justified in writing and applied consistently throughout the project.

Prediction Task

The prediction task is formulated as a binary classification problem with spatial and temporal structure.

  • Outcome
    For each spatial unit and month, define a binary variable indicating whether at least one notable wildfire occurs during that month.

  • Spatial unit
    You may choose an appropriate spatial unit (e.g., grid cells, counties, or other geographic partitions). The choice should balance spatial resolution, interpretability, and data sparsity.

  • Temporal unit
    Month. You may refine your resolution to week or day with clear justification.

  • Predictors
    You may select any variables in the dataset as your predictors. External data sources (e.g., weather or land cover) are allowed but not required.

##Defining Binary Outcome Variable The outcome of the prediction task is defined as a binary variable that reveals whether a notable fire occured at a certain month. The binary variable can be defined as Y = 1 if there is one notable wildfire that occurs in a state during that month and Y = 0 otherwise. This binary variable helps identify the occurrence and frequency of wildfire activity in a certain state at a certain month instead of predicting the number of fires taking place.

##Constructing predictors The initial wildfire data set contained records about individual fire events, but the prediction model must account for incidence and location of notable wildfires. Therefore, the predictor variables were constructed by aggregating wildfire attributes for each state-month unit. The predictors that will best predict the outcome include the number of fires (fire_total), total area burned by wildfires (total_area), and average size of wildfire (avg_size). fire_total was constructed by counting the number of rows with notable wildfires (> 300 acres) that happened in the same state and month. total_area is the sum of all fire sizes that occurred for the same state and month pair while avg_size is the average wildfire size for the same state and month pair. These variables effectively encapsulate and summarize the location, severity, and frequency of wildfire activity within each state for a specific month. They will also be meaningful predictors when modeling wildfire incidences to determine the probability of a wildfire occurring for a certain geographic area during a future month.

##Missing Data and Edge Cases In order to handle the missing data and edge cases, the missing values for FIRE_SIZE are handled by removing all of them when calculating the total and average burned area so that these NA values did not impact the aggregation of predictors (na.rm = TRUE). Moreover, the STATE variable will have missing values if a state does not have any notable fires. These rows in the data set will be removed so those states will not be accounted for when predicting. When combining the state and month pairs, if there were outcomes where there were no notable fires that occurred, then it was assigned the binary value of 0. This ensures that there are cases with both notable and non-notable fires that occur for a certain state for a given month. Overall, the data set only contains minimum missing or null values, so they were handled according to the value that was missing.
